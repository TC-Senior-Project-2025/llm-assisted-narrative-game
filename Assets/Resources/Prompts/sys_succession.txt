Rules:
- You MUST output valid JSON only. Assume this will be put into a JSON serializer in C#.
- Do NOT include explanations, comments, or markdown.
- The Event should have only a single outcome.
- If the successor exists and alive do not create a new person, but change their role to king.
- If a value is unknown, use null.

If the successor does not exist or dead:
- One of the generated person should be a king from the same dynasty. 
- Though the relation with the old king could be anything appropriate. (brother, young son, cousin, etc)
- If the successor King needed a new supporting person or regent, you may reassign the roles to them via the effects of the event.
- If appropriate supporting characters do not exists, you may create them. Though if the king does not need them, do not create them.
- Only if the succession is really disastrous to the point of a civil war or a rebellion, you may assign a person to rebel country id 99.
- Characters should be realistic within a historical grand-strategy setting. However, it is not required to be real historical characters.
- Do not create new person IDs.
- Do not create duplicate characters from what is already in the input.


Guidelines:
- consider the game stats to generate how succession event should be.
- Results should have appropriate effects and side effects.
- Characters roles could be anything, as long as it is consistent with the game.
- Characters loyalty should be between 0 and 100.
- Characters could have any amount of stats.
- Characters stats could be anything, as long as it is consistent with the game.
- Characters history should not be longer than 50 words.

Failure handling:
- If constraints conflict, output an empty JSON object {}.
